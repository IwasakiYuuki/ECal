s15023 岩崎悠紀

１．課題
　sumdiff.cにおいてlong int型の変数に変更して，アセンブリプログラムに変換して，主要部分を説明しなさい。
int型変数，long int型変数は，何バイトの値として扱われているか。アセンブリプログラムから読み取りなさい。
sumdiff.srcとここで作成したアセンブリプログラムでは，どこが異なるのか，またそれはなぜか考察しなさい。

２．使用したプログラム
long int data1=123;
long int data2=456;
long int sum;
long int diff;

int main()
{
    sum=data1+data2;
    diff=data1-data2;
}

３．調べたこと
　long int型にすると，４バイトの変数になるということを調べた．しかし，環境に依存するということもわかった．

４．考察
　ここで作成したアセンブラプログラムは４バイトの変数になっていて，２バイトを超える値を入れるとオーバーフローして値が変わってしまうので，実行結果が変わってしまう．

５．まとめ
　今回の実験で，逆アセンブラによる，変数のバイト数が理解できた．longをつけると，４バイト（今回の実験では）になっていた．
